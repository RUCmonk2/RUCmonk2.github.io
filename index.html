Useful Tips

Command Line

用 GitHub CLI 管理仓库

GitHub CLI 把创建仓库、克隆仓库、查看状态和推送修改这些操作搬到终端里。配置好登录之后，我可以用命令完成原本需要在网页上点来点去的流程，也方便让 Codex 在授权范围内直接协助创建和管理项目。

官方手册 gh repo create gh repo clone

AI Workflow

把 Codex 交接机制做成项目模板

我最早是在科研文件夹里放交接 README，用来记录当前主线、重要文件、遗留问题和可复制的开场白。后来又给项目加入 temps 临时区，用来收纳待整理资料。再进一步，我把这套做法抽象成通用指南。这里有些内容不是为了让读者逐字读懂，而是给 Codex 执行的：如果你也想要类似效果，可以把指南网址直接发给 Codex。

公开指南 在线阅读 Markdown 原文 temps 临时区 项目模板

Prompting

让 AI 先复述需求和计划

在让 AI 改文件、写代码或 push 之前，先要求它复述我的需求，并说明准备怎么做。这样可以提前发现理解偏差，把模糊想法整理成明确任务，也能降低改错文件、放错位置或直接误操作的风险。

需求澄清 执行前对齐 降低误操作
