Useful Tips

实用小技巧

Command Line

用 GitHub CLI 管理仓库

GitHub CLI 把创建仓库、克隆仓库、查看状态和推送修改这些操作搬到终端里。配置好登录之后，我可以用命令完成原本需要在网页上点来点去的流程，也方便让 Codex 在授权范围内直接协助创建和管理项目。

官方手册 gh repo create gh repo clone

AI Workflow

用交接 README 维持 Codex 长线任务上下文

长线任务容易在多次对话之间丢失上下文。我会在项目目录里放一个固定的交接 README，记录当前主线、重要文件、遗留问题和可复制的开场白。每次新开 Codex 窗口时，先让它阅读这个入口文件，再继续推进任务。

在线阅读 Markdown 原文 目录快照

Prompting

让 AI 先复述需求和计划

在让 AI 改文件、写代码或 push 之前，先要求它复述我的需求，并说明准备怎么做。这样可以提前发现理解偏差，把模糊想法整理成明确任务，也能降低改错文件、放错位置或直接误操作的风险。

需求澄清 执行前对齐 降低误操作
