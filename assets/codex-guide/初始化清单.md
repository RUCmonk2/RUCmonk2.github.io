# 初始化清单

最后更新：2026-07-09

## 适用场景

用户新建了一个项目、资料文件夹、课程文件夹、科研文件夹或网站仓库，并希望以后 Codex 能跨窗口延续上下文。

## 执行前检查

1. 确认目标项目路径。
2. 检查目标项目是否存在。
3. 检查是否已有 `temps/`、`codex交互reademe.md`、`codex语境交接/`。
4. 如果已有同名文件，先读取并说明差异，不直接覆盖。
5. 如果目标项目是 Git 仓库，运行 `git status --short`，避免混入无关改动。

## 标准创建内容

1. 创建 `temps/`。
2. 创建 `temps/.gitignore`：

```gitignore
*
!.gitignore
!.gitkeep
!README.md
```

3. 创建 `temps/.gitkeep`。
4. 创建 `temps/README.md`，记录临时资料使用规则、当前清单、建议清理节奏和清理记录。
5. 创建 `codex交互reademe.md`。
6. 创建 `codex语境交接/`。
7. 创建以下交接文件：
   - `CURRENT.md`
   - `OPEN_QUESTIONS.md`
   - `FILE_MAP.md`
   - `DECISIONS.md`
   - `JOURNAL.md`
   - `PROMPTS.md`

## 写入内容要求

- 所有路径必须改成目标项目真实路径。
- `CURRENT.md` 要写当前项目主线，而不是泛泛而谈。
- `FILE_MAP.md` 要根据目标项目已有目录写，不要只复制模板。
- `OPEN_QUESTIONS.md` 可以先写少量真实问题；如果项目刚开始，就写“待用户补充目标”。
- `DECISIONS.md` 至少记录 `temps/` 规则和 Codex 交接机制。
- `temps/README.md` 要根据目标项目写清临时资料类型、清理阈值、当前清单和清理记录；不要只留空模板。
- `JOURNAL.md` 要记录本次初始化动作。
- `PROMPTS.md` 要放可直接复制的提示词，路径指向目标项目。

## 处理 temps 的标准流程

1. 列出 `temps/` 文件。
2. 读取 `temps/README.md`，核对当前清单是否过期。
3. 判断每个文件的类型和用途。
4. 给出处理方案：
   - 移动到正式目录；
   - 提取信息写入正式文件；
   - 只做参考后删除；
   - 暂时保留等待用户确认。
5. 执行前如涉及删除或覆盖，确认用户意图是否明确。
6. 执行后更新 `temps/README.md` 的当前清单和清理记录。
7. 如果资料被移动到正式目录、提取进正式文件，或影响项目主线，再更新 `JOURNAL.md` 和必要的 `FILE_MAP.md`、`CURRENT.md`、`OPEN_QUESTIONS.md`。

## 完成后检查

1. 列出新增文件。
2. 如果是 Git 仓库，检查 `git status --short --untracked-files=all`。
3. 检查 `temps/.gitignore` 是否能忽略普通临时文件，但保留 `.gitignore`、`.gitkeep` 和 `README.md`。
4. 向用户说明：
   - 创建了什么；
   - 哪些文件是入口；
   - 以后新窗口该如何引用；
   - 是否有未提交的新增文件。
